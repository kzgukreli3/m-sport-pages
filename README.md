# m-sport-pages

本仓库用于归档和发布多个独立的 HTML 页面。这些页面通常用于展示、预览、测试或临时分享前端内容，不针对任何特定网站或域名。

## 项目简介

m-sport-pages 是一个静态 HTML 页面集合仓库。每个页面均为独立、自包含的 HTML 文件，可以直接通过浏览器打开，或通过 GitHub Pages 等服务进行访问。

仓库内容涵盖多种用途的页面，例如信息展示、功能演示、样式测试等。所有页面均保持轻量、简洁，不依赖外部复杂框架。

## 目录说明

```
m-sport-pages/
├── index.html          # 仓库首页 / 导航页
├── pages/              # 存放独立 HTML 页面
│   ├── example1.html
│   ├── example2.html
│   └── ...
└── assets/             # 公共资源（如有需要）
    ├── css/
    ├── js/
    └── images/
```

- `pages/`：主要页面存放目录，每个文件为一个独立页面。
- `assets/`：存放多个页面可能共用的样式、脚本或图片资源（可选）。
- `index.html`：可选的仓库首页，用于列出或导航到各个页面。

## 页面归档说明

- 每个页面文件均为完整、独立的 HTML，包含必要的样式与脚本。
- 页面命名建议使用有意义的英文或拼音，避免特殊字符。
- 页面内容仅用于归档展示，不包含任何商业推广或特定网站链接。
- 如有更新，建议保留旧版本或单独标记版本号，便于回溯。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接在浏览器中打开 `pages/` 目录下的任一 HTML 文件。
3. 若启用 GitHub Pages，可通过仓库设置中的 Pages 功能访问根目录或指定目录下的页面。

## 维护说明

- 欢迎提交 Pull Request 添加新的页面或改进现有页面。
- 请确保新页面符合本仓库的归档定位：独立、自包含、无外部依赖（或仅依赖本仓库 assets 目录下的资源）。
- 不建议在页面中加入跟踪、统计、广告等第三方脚本。
- 如有大规模结构调整，建议先通过 Issue 讨论。
